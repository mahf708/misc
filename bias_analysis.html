How to Use This Tool

What is reviewer bias?

Different reviewers have different scoring tendencies. A strict reviewer might average 22/30 while a lenient one averages 26/30. This tool corrects for these systematic differences so all applicants are evaluated fairly regardless of which reviewers they were assigned.

Reading the scatter plot

X-axis: Raw average score (before adjustment). Y-axis: Adjusted score. Points above the diagonal line benefited from adjustment (had stricter reviewers); points below were reduced (had more lenient reviewers). The orange lines mark your cutoff threshold.

Color coding

Green: Passes both raw and adjusted thresholds (strongest candidates). Blue: Passes raw only (may have had lenient reviewers). Purple: Passes adjusted only (may have had strict reviewers). Gray: Neither.

Selection strategies

Raw Only: Use original scores as-is. Union: Include anyone passing either method (most inclusive, catches strong candidates penalized by strict reviewers). Both Methods: Require passing both (conservative, highest confidence).

Adjustment Methods Explained

None (No Adjustment)

Uses raw scores without any correction. All points appear on the diagonal since raw = adjusted. Use when: You trust all reviewers are equally calibrated, or as a baseline to compare against adjusted methods.

Bias Adjust (Mean Centering)

Subtracts each reviewer's average deviation from the global mean. If a reviewer averages 2 points below average, their scores get +2. Best for: Correcting consistent over/under-scoring. Limitation: Doesn't account for variance differences.

Z-Score Normalization

Standardizes each score relative to the reviewer's own mean and standard deviation, then maps to the global scale. Best for: Reviewers with very different distributions (some spread scores widely, others cluster). Limitation: Can amplify noise for reviewers with few reviews.

Percentile Normalization

Converts each score to a percentile rank within that reviewer's distribution, then maps to the equivalent percentile in the global distribution. Best for: Non-linear reviewer behavior (e.g., never gives 30s). Limitation: Less precise with few reviews per reviewer.
